## Stack & hard conventions
- **React + Vite + TypeScript.**
- **DaisyUI only** for UI (theme locked to `dracula`). No shadcn, no Radix, **no hand-written utility CSS for our own components** — use DaisyUI + Tailwind utilities (incl. responsive `sm:` breakpoints) only. `src/index.css` is the Tailwind import, DaisyUI plugin, font tokens, body font — **plus a few necessary vendor overrides**: YARL caption text-selection, Leaflet popup theming, and the `journey-route` marching-dash keyframes. Keep additions to vendor-override/animation only.
- **Image viewer:** `yet-another-react-lightbox` (YARL) with Thumbnails, Captions, Zoom, Fullscreen plugins. Do NOT rebuild a carousel by hand — that caused many bugs.
- **Interactive maps:** `leaflet` + `react-leaflet` + `leaflet-minimap` (overview navigator) power two features, both code-split & lazy-loaded so Leaflet stays out of the initial bundle: the "Journey of Odysseus" (`src/JourneyMap.tsx`, CRS.Simple over a single `ImageOverlay`) and the "Atlas" (`src/AtlasMap.tsx`, CRS.Simple over a Cloudinary-hosted `TileLayer` pyramid — see "Atlas map" below for why it needs tiles instead of one image).
- **Fonts:** `Cinzel` (serif) for the hero masthead ONLY; `Hanken Grotesk` (sans) everywhere else; `Noto Sans SC` for Chinese. Loaded async in `index.html`; Cinzel latin woff2 is preloaded.
- Most logic lives in `src/App.tsx`; the two interactive maps are `src/JourneyMap.tsx` and `src/AtlasMap.tsx`.

## Data model
- `src/data/glossary.json` — array of entries: `{ term, pron, tag, def, zhName, zhPinyin, zhDef, art: string[] }`. `art` is an ordered list of keys into art.json; `art[0]` is the card cover.
- `src/data/art.json` — `{ key: { file, artist, title, year, source, license?, cld } }`. `cld` is the Cloudinary public_id (delivery). `file` (`/art/<key>.jpg`) is a legacy/fallback path only. `license` is only set for non-PD (CC) images.
- `categoryOf(tag)` maps tags → 4 filter categories: **gods** (god\*), **monsters** (monster), **mortals** (person/hero/people/animal), **world** (place/thing/idea/trick). Filter labels: Gods · Mortals · Monsters · **World**.

## Captions / licensing
- Caption (see `slides` in `App.tsx`): the credit (`Artist · Title, year`) and the optional `note` go in the YARL slide **`description`** (credit line, then `<br/>`, then note), and the slide **`title` is left empty on purpose** — YARL renders `title` in the top toolbar with `white-space:nowrap;text-overflow:ellipsis`, so it truncated on mobile. `description` wraps (set `descriptionMaxLines:5`). Graceful omission throughout.
- **Licence is NOT shown per-image** (was removed from the caption line) — the footer's generic "public domain or openly licensed (CC)" covers attribution. `license` stays in `art.json` for record only.
- **Metadata fill (done):** every non-map artwork now has an `artist` (real painter/vase-painter name, or `"Anonymous"` for anonymous ancient works) and a `year` (filled from each Commons source's *object* date — NEVER the photo/upload timestamp, which is the trap; circa dates where uncertain). `captions.csv` (repo root) is a one-row-per-image export for eyeballing.
- The `note` is a short subject locator/clarifier — locate the subject in a multi-figure work ("Ajax — the warrior at right (inscribed AIANTOS)") or explain an indirect-but-valid connection ("The Titans — Cronus's kin — cast down by the Olympians"). Written by viewing each image (vision pass); conservative (blank when single-figure/obvious/uncertain). **282 of 500** images have a `note`; **105** are CC-licensed. **English titles only** — foreign/messy/catalogue titles were stripped. **All** titles (harvested + originals) were standardised: translated to English, museum/inventory/accession noise removed, truncations/"artist-as-title"/book-scan prefixes fixed, Title Case; `artist` fields scrubbed of Commons uploader/photographer handles (kept real painters + vase-painter conventions; blanked anonymous ancient works). Re-standardise from the `source` filename (ground truth) — the stored `title` was sometimes lossy/truncated.
- **Places & maps:** all 84 places (from Wilson's glossary) are `tag: "place"` → **World** filter. Each is illustrated by one of **6 antique PD maps** (Ortelius *Vlyssis Errores* / Wanderings of Ulysses & *Aegyptus Antiqua*, Lapie *Voyages*, *Graecia Homerica*, Delisle *Northern Greece*, *The World according to Homer*), `public/art/map-*.jpg`. A map is **shared across the many places it contains**: each place has its own art record keyed `<slug>-map` that reuses the map's `cld` but carries its own locating `note` ("Chios — an Aegean island off the coast of Asia Minor"). So maps are the ONE deliberate exception to image-uniqueness — exclude `*-map` keys from both the source-dedup and perceptual-dup checks.
- **Vlyssis Errores fix (done):** the `map-wanderings.jpg` ("Wanderings of Ulysses") file was originally **mislabelled** — it held Ortelius's *Red Sea* (Erythraean) plate, not the Mediterranean Ulysses map. Replaced with the **genuine *Vlyssis Errores* (1597) inset**, cropped from a 13238px PD scan of the Red Sea plate (the inset lives in its lower half; user hand-cropped the final framing). This same image is the **base of the Journey map** below. The 16 wandering-isle places (`*-map` keys sharing its `cld`) now show the correct map.
- **Perceptual-dup check (tool):** beyond source-filename dedup, the same artwork sometimes exists as two different files (low-res caption-baked vs clean high-res). Detect with a PIL dHash+aHash sweep over `public/art/*.jpg` (restrict to live art.json keys, skip `map-*`); pairs with dHash≤12 & aHash≤18 are candidates — VIEW a side-by-side montage to confirm (dark images throw false positives), keep the higher-res/caption-free copy. Re-run after big harvests.
- **Dedup invariant:** every NON-MAP image appears once. Enforced by grouping on the `source` Commons filename and keeping a single key per group, **preferring the one that is some entry's cover** so no card cover is orphaned. Re-run after any harvest. Note: deduped images may remain as orphan assets in the Cloudinary `odyssey` folder (delete separately).
- Footer says "public domain or openly licensed (CC)". Sourced from Wikimedia Commons. Accept PD / CC0 / CC-BY / CC-BY-SA only.

## Working with artwork (lessons learned)
- Fetch scripts query the Commons API (`generator=search`, `prop=imageinfo`, `iiprop=url|mime|extmetadata`). **Filter aggressively:** skip `.pdf/.djvu/.svg`, book scans (`(IA `, "Adventures of Ulysses"), maps, logos, and species/astronomy named after Greek figures (butterflies like *Morpho menelaus* / *Eumaeus atala*, planet "Saturn with auroras"), film stills, perfumery signs, "goddess card" composites.
- After fetching: build a contact-sheet montage with ImageMagick and eyeball it; check for cross-language duplicate works (same painting filed under English + French/German/Dutch names — dedupe, keep English).
- A greedy "cover diversification" pass reorders each entry's `art` so covers are unique where alternatives exist.
- **Full-article harvest (done):** pulled *every* PD/CC image off each entry's en.wikipedia article (`generator=images`), keyed `<slug>-w<NN>`. This sweeps in junk that `generator=images` can't pre-filter — maps, coins, site/building/landscape photos, Britannica diagrams, book-plate scans, museum replicas, ritual photos, a fossil skull, and **off-subject gallery/navbox thumbnails** (e.g. Bernini's *Apollo & Daphne* landed on Ajax). The original Commons filename lives in the `source` field (the on-disk `file` is renamed) — QA by dumping `source` basenames + a contact-sheet montage, then prune. 32 were pruned this round.
- **Image weight:** local masters in `public/art` are recompressed to `-resize '1600x1600>' -quality 82 -interlace Plane -sampling-factor 4:2:0 -strip` before upload.
- **Relevance + caption audit (done):** a vision pass over *every* image (subagents that actually open each `public/art/<key>.jpg`) checking (a) is it on-subject for its entry, (b) is the caption helpful. `generator=images` pulls tangential article images that *look* fine by filename but are off-subject — e.g. Blake's *Comus* on Circe, *School of Athens* on hospitality, a "Marlowe" portrait on Helen, a Theseus/Minotaur cup on Hades. **Removed off-subject; never trust the title alone — verify by viewing.** Also killed unusable images (dense scholarly "Nekyia reconstruction" you can't zoom) and same-scene duplicates (kept the clearer one — e.g. Lykaon Painter vase over van Thulden's *Burning of Elpenor*). Be conservative: keep if it plausibly relates; a clarifying `note` often beats removal for indirect connections.
- **Ajax is two people:** "Ajax the Great" (Telamonian; Book 11 underworld shade) and "Ajax the Lesser" (Locrian, son of Oïleus; Book 4 drowning, the Cassandra outrage) are separate entries. Don't mix their art.

## Journey of Odysseus map (`src/JourneyMap.tsx`)
- Leaflet **CRS.Simple** image map over the *Vlyssis Errores* base (Cloudinary, **version-pinned** delivery URL `…/w_3000/v<ver>/<cld>` to bust browser cache). Coordinate space = the master's pixel size; `const W=4000, H=2337`. `yx(x,y)=[H-y,x]` flips y for CRS.Simple. Bump `MAP_VER` whenever the base image is re-uploaded.
- **15 voyage stops** (`STOPS`, 14 canonical per Stephen Fry's "Journey of Odysseus" plus one extra) hand-calibrated onto Ortelius's own labels (Troy/Ilium, Cicones/Ismarus, Aeaea/Circe, Cimmerii/Hades, etc.). Each links to its glossary `term`. The extra stop is Odysseus's second landing on Aeaea (Book 12: he actually beaches the ship and stays the night to bury Elpenor and get Circe's final sailing directions, not just sail past) — it reuses `term: "Circe"` (same island, same glossary entry/artworks) so it needed its own React `key` (`s.n`, not `s.term`, since two stops now share a term).
- **Narrative sea route** in `LEG_VIAS[i]` (per-leg bend points) — faithful to Homer: round Cape Malea to the Lotus-Eaters, the western-Ocean dip to Hades and **back to Aeaea** (Circe visited twice), the **Scylla/Charybdis strait threaded twice** (out + the wreck-driven double-back). Smoothed with **centripetal Catmull-Rom** (`smooth()` — centripetal, not uniform, to avoid overshoot on uneven spacing; emits `ROUTE_SEG` points per control-segment, so input point `k` lands at smoothed index `k*ROUTE_SEG`). Drawn as two polylines: a white **casing** under the purple. **Gotcha:** the casing must use the **same `journey-route` dash+animation** as the purple — a *solid* casing fills the dash gaps and kills the marching-ants effect.
- Features: guided **tour**, zoom-aware place-name labels, hover tooltips, **click-popup** with bilingual entry info + "View artworks" (`lookup` prop from App resolves term→entry; the map stays open under the lightbox so closing returns to it), an overview **navigator** (`leaflet-minimap`), and a Fry-style legend.
- **Tour camera** (`TourController` + `panAlong`): between *adjacent* stops the camera **glides ALONG the route curve at a fixed `TOUR_ZOOM`** (rAF tween over the curve slice `routeLatLng[stopRouteIdx[from]..stopRouteIdx[to]]`, speed scaled to leg pixel-length, clamped) — NOT `flyTo` (which zooms out-then-in and feels disconnected). Non-adjacent jumps and the first stop ease straight in; idle (`idx<0`) zooms out to the whole map. Glide pace is by **on-screen pixels** (`glideMs(px)` = `TOUR_PPS` 150 px/s, clamped `GLIDE_MIN`/`GLIDE_MAX` 3.5–11s) — a sail, not a whip. **Cadence is decoupled, not a fixed `STOP_MS`:** each stop holds for `cycleMs = glide-in + DWELL_MS` (3.5s), so the huge Cicones→Lotus-Eaters leg (~6800px screen, vs ~700px typical) can glide its full ~11s without starving the dwell on short stops. `legGlideMs(from,to)` measures the route slice the same way `panAlong` does (px·2^TOUR_ZOOM), so timer + camera agree; both the auto-advance `setTimeout` and the **countdown bar** (`tour-progress` keyframe, `key`'d per stop) use `cycleMs`. A tiny antique-galley **ship** (`shipIcon` divIcon) rides the route during the tour — `panAlong`'s `onMove` callback `setLatLng`s it each frame; since the camera centers on that same point it sits at screen-center while the map scrolls (hidden via `opacity:0` when idle). Legend clicks drive a separate `focus` state that pans the camera through the same `TourController` while **keeping the legend open** — they don't start the tour or open artwork.
- **URL route:** `#journey` opens the map (App reads `window.location.hash`); `#humaneyeball` opens it in **calibration mode** — drag pins, click the route to add a sea-bend, drag/dbl-click bends, then "Copy coordinates" to get the `{stops, legVias}` JSON to paste back into the source.
- **Mobile:** modal goes full-screen, the wide map **covers** the screen (min zoom = `getBoundsZoom(bounds, true)`) and pans horizontally (follows the east→west voyage); opens focused on Troy; legend hidden, tour controls live in the tour card.
- **Calibration workflow:** to reposition pins/route, open `#humaneyeball`, drag, copy the JSON, and bake values into `STOPS` / `LEG_VIAS` (don't hand-guess coordinates). To locate a label on the map, crop the master with PIL + a coordinate grid and view it. The floating "Calibration" panel (live JSON dump + Copy button) is the shared `src/CalibrationPanel.tsx` — see its own section below.
- **react-leaflet `pathOptions.className` is dropped in the PRODUCTION build (works in dev) — a real trap.** `createPathComponent` calls `new L.Polyline(positions, options)` with `className` nested inside `pathOptions`, but Leaflet only reads a *top-level* `options.className` in `_initPath`, and react-leaflet applies pathOptions via `setStyle()` which **never touches className**. So the class is never put on the SVG path. It *appears* to work in `vite` dev ONLY because StrictMode's double-mount re-runs `_initPath` after `setStyle` happened to merge className into the options — a fluke absent from the single-mount prod bundle. Symptom this session: the `journey-route` marching-ants class (hence the animation) vanished in production while fine on localhost. **Fix: apply the class yourself on the layer `add` event** (`eventHandlers={{ add: e => e.target.getElement()?.classList.add("journey-route") }}` → `markRoute`), not via `pathOptions.className`.
- **Dev/testing gotchas:** ports **5173/5174 are usually taken** by sibling projects (e.g. *Chord Drill*) — the dev server lands on a higher port; check the Vite log for the real one. **Dev≠prod for the map:** Vite serves CSS unminified and React in StrictMode, so the className bug above and any minifier CSS rewrite hide in dev — reproduce production issues with `npm run build` + `vite preview`. The `chrome-devtools` MCP refuses to attach if a Chrome with its profile is already open; fall back to driving the live/preview URL with **Playwright WebKit** (Safari engine — the iPhone test target) via the npx-cached `playwright` (scripts must live in that dir to resolve the module). When testing the **tour/animated state**, same-URL `#journey` navigation does **not** reset React state, and multiple open tabs each run their own timers — both cause misleading "advancing too fast" readings. Force a fresh mount (navigate to `/` then `#journey`, or a new tab) and verify intervals with `console.log(performance.now())`. Cadence is **per-leg `cycleMs`** (glide-in + `DWELL_MS`), not a single fixed value.

## Atlas map (`src/AtlasMap.tsx`)
- **Config-driven, multi-plate** (mirrors `JourneyMap`'s `src/data/journeys/` pattern): `src/data/plates/{types,index,<slug>}.ts` is the registry — `types.ts` defines `AtlasPlace` (`term`/`label?`/`noGloss?`/`x`/`y`) and `PlateConfig` (`slug`/`title`/`w`/`h`/`maxZoom`/`tileBase`/`attribution`/`places`); `index.ts` exports `PLATES: Record<string, PlateConfig>` + `DEFAULT_PLATE_SLUG` (currently `"graecia"`). `AtlasMap.tsx` itself takes `config` + `editing` props (no module-level constants) and renders a `<select>` plate switcher in the modal header.
- **6 plates**, all Ortelius (Theatrum Orbis Terrarum): `rubri` (*Erythraei Sive Rubri Maris Periplus*, 1597 — the original full-Red-Sea plate: Egypt/Arabia/Persia/India/East Africa plus the cramped Aegean *Vlyssis Errores* inset that the Journey map also uses; 13238×10802px, maxZoom 6, ~3010 tiles), `graecia` (*Graecia Sophiani* — the primary Greek map, mainland Greece/Peloponnese/Crete/Aegean at real cartographic scale; 12325×8913, maxZoom 6, ~2334 tiles), `aegyptus` (*Aegyptus Antiqua*, 6498×9896, maxZoom 6, ~1373 tiles), `natoliae` (*Natoliae... Asia Minor*, 5560×7725, maxZoom 5, ~923 tiles), `palestinae` (*Palestinae sive... Terrae Promissionis*, 8000×6144, maxZoom 5, ~1025 tiles), `africae` (*Africae Tabula Nova*, 12504×9430, maxZoom 6, ~2470 tiles). Masters/tiles live in git-ignored `plates/<slug>/` (not `public/art/` — too large for the `cld sync` artwork pipeline).
- **Why tile pyramids, not `ImageOverlay`s like JourneyMap:** each master is 60–140MP, far past what Cloudinary's live `c_crop` transform API can crop in one request. Sliced offline per plate: `vips dzsave <master> tiles --layout google --suffix '.jpg[Q=82]' --tile-size 256 --overlap 0` (via `scripts/make-plate.sh <slug> "<Commons file title>"`, which also curls the master from `https://commons.wikimedia.org/wiki/Special:FilePath/<title>`).
- **Hosting:** each tile uploaded to Cloudinary at a **deterministic** `public_id` — legacy `rubri` at bare `atlas/{z}/{row}/{col}`, every new plate at `atlas/<slug>/{z}/{row}/{col}` (`config.tileBase` picks the prefix) — via `scripts/upload_tiles.py <slug>` (`+ --verify` to sample delivery URLs incl. corners), using the `cloudinary` SDK directly (NOT the `cld sync` CLI) with `public_id=...`, `overwrite=True`, `unique_filename=False`. Deterministic public_ids mean tile URLs need **no lookup manifest**: `${CLD}/atlas[/<slug>]/{z}/{y}/{x}` (no `f_auto,q_auto` — see cache-fragmentation note in `AtlasMap.tsx`).
- **Gotcha — tile addressing is TRANSPOSED:** `vips dzsave`'s "google" layout writes tiles on disk as `{z}/{row}/{col}.jpg` — row is the directory, col is the filename — the transpose of Leaflet's `{z}/{x}/{y}` template (x=column, y=row). Get this backwards and the map renders as a **scrambled, transposed puzzle** (each tile intact, wrong tile in each cell) — NOT a blank/missing-tile failure, which makes it a confusing bug to spot. Fix: swap y/x in the URL template (`atlas/{z}/{y}/{x}`, not `atlas/{z}/{x}/{y}`).
- **Gotcha — `L.CRS.Simple.unproject(point, zoom)` called STATICALLY silently skips the scale division** (it just applies the axis-flip transform to whatever raw point you give it) — only the LIVE MAP INSTANCE's `map.unproject(point, zoom)` correctly divides by `scale(zoom)=2^zoom` first. Bounds/marker math built from the static call end up in raw pixel units, and `fitBounds` "solves" the resulting nonsense box by landing on a wrong zoom, which makes the tile grid compute negative row/col indices (a wall of 404s like `atlas/2/-2/1`). **Always compute bounds/positions via the live map's instance methods** (`unprojectPixel(map, x, y, maxZoom)` helper in `AtlasMap.tsx`), never the static `L.CRS.Simple` object.
- **Modal-mount timing:** react-leaflet's initial `bounds`-prop fit runs before the modal's CSS layout has settled (container still 0×0) — fixed with a `ResizeObserver` (`FitWhenReady`) that fits once the container has real dimensions. Must fit **only once** (guarded by a `fitted` flag) — re-running `fitBounds` on every observed resize (Leaflet's own zoom-animation causes micro-reflows the observer also sees) snapped the view back to "fit all" on every scroll-zoom tick, making scroll-to-zoom appear completely broken. Same one-shot idea as `JourneyMap`'s `LockMinZoom` (`focused` flag). Switching plates remounts `AtlasMap` via `key={slug}` so this and tile-prefetch state reset cleanly.
- **Mobile "cover" fit:** same fix as `JourneyMap`'s `LockMinZoom` — `map.getBoundsZoom(bounds, true)` as the min zoom, so the map always fills the frame instead of "fit inside" leaving grey letterbox bands on a portrait phone (caught only by testing on an actual iPhone SE simulator, not visible on desktop where the container is aspect-locked to the image via `sm:aspect-[...]`).
- **Calibration:** `#atlas-eyeball` (default plate) or `#atlas/<slug>-eyeball` (any plate) — Shift+click the map to drop a new pin (named "untitled", rename in the footer panel), **drag existing pins** to adjust (`dragend` converts the marker's `getLatLng()` back to pixel coords via `map.project()`), then "Copy all" in the `CalibrationFooter` (search → crosshair-focus → drag → copy) to get the plate's JSON to paste back into its `src/data/plates/<slug>.ts` file. **Pin drag must stay Leaflet-NATIVE (`draggable` prop + `dragend`)** — a hand-rolled rewrite cost 6 failed commits: Leaflet markers never fire `pointerdown`/`pointermove` (Leaflet translates pointer events to `mousedown`/`mousemove` before dispatch), and "Shift+drag the pin" can't work either (Shift+drag is Leaflet's BoxZoom gesture at the container level, and `L.Draggable` ignores shift-mousedowns anyway — also no Shift key on iOS). Pin-drop requires **Shift+click** (plain clicks — including both clicks of a double-click-to-zoom — used to litter stray "untitled" pins); that makes adding pins desktop-only, since iOS has no Shift key.
- **Pin hit target:** the visible dot is 24px (`size-6`), but `pinIcon`'s actual `divIcon` is 40px (`iconSize:[40,40]`, dot centered inside via a padded wrapper) with `cursor-grab`/`active:cursor-grabbing` — a slightly imprecise mousedown still grabs the marker instead of falling through to the map's own click handler, which in eyeball mode means "drop a stray new pin", not "drag this one".
- **Pins:** `term` must exactly match the glossary term (or carry `noGloss: true` for deliberate poem-text/label-only pins) or the popup's "View artworks" lookup silently fails (bit us twice: "Mount Olympus" vs glossary "Olympus"; a whole batch-rename in commit `2c4afef`). `label?` supplies a shorter display name without touching the lookup key (e.g. `term: "Argos (the city)"`, `label: "Argos"`, to disambiguate from the dog's entry). **`npm run check:pins`** (`scripts/check-pins.ts`, tsx) guards term validity, plate pixel bounds, per-plate duplicate terms, journey-stop terms, and glossary place coverage (only `Ocean` is deliberately unpinned) — run after every pin or plate change. `PLACES.md` is the regenerated term×plate coverage matrix (source of truth for pin data is each plate file, not the doc).
- **Wheel-zoom sensitivity:** `wheelPxPerZoomLevel` on both `AtlasMap` and `JourneyMap` is `15` (down from a sluggish default `40`) — lower is more sensitive/faster zoom per scroll tick.
- **Entry point / hash routing** (mirrors `JourneyMap`'s `#journey/<slug>` pattern): `#atlas` / `#atlas-eyeball` open the `DEFAULT_PLATE_SLUG` plate; `#atlas/<slug>` / `#atlas/<slug>-eyeball` open any other plate. `App.tsx`'s `parseAtlasHash` regex is kept distinct from the journey one so `#atlas-eyeball` doesn't collide with `#humaneyeball`.
- **Not yet done:** Graecia's pins are still an uncalibrated seed grid and `rubri`'s redundant Greek pins haven't been removed — see the IN-FLIGHT note at the top of this file; no minimap-size tuning for mobile (fixed 132px width, same as JourneyMap); no drag-declutter pass for dense pin clusters at low zoom.

## Shared calibration UI (`src/CalibrationPanel.tsx`)
- Both eyeball modes (`#humaneyeball`, `#atlas-eyeball`) render the same floating "Calibration" panel — title, hint text, a live-updating readonly JSON `dump` textarea, and a "Copy coordinates" button. Extracted into one shared component so this markup exists in exactly one place; the two maps' actual pin/route-dragging *logic* stays separate and is NOT shared, because the underlying coordinate math genuinely differs: `JourneyMap` converts via a plain `yx(x,y)=[H-y,x]` flip against its single `ImageOverlay`, while `AtlasMap` must go through the live map instance's `project`/`unproject` because its tiled `CRS.Simple` layer needs the `scale(zoom)=2^zoom` division a plain flip doesn't do (see the Atlas gotcha above). `JourneyMap` also has route-bend calibration (click-to-add/drag/dblclick-remove on the sea route) that Atlas has no equivalent of. Forcing those into one abstraction would either break Atlas's zoom correctness or bolt unused route machinery onto it, so only the panel *shell* is shared.
- **Portalled to `document.body`, not rendered as a child of `modal-box`, and it's load-bearing, not cosmetic.** DaisyUI's `.modal-box` sets `translate: 0; scale: 1` unconditionally (even at the identity values) — and per the CSS spec, ANY non-`none` value of `translate`/`scale`/`rotate`/`transform`/`filter` on an ancestor makes that ancestor the *containing block* for `position: fixed` descendants. A panel left inside `modal-box` with `position: fixed` would therefore only be draggable within the modal card's own box, not the actual browser viewport — `createPortal(..., document.body)` sidesteps that trap so the panel can genuinely be dragged anywhere on screen, including outside the visible map card.
- **Draggable via native `PointerEvent`s** (`pointerdown` on the "⠿ Calibration" header, `pointermove`/`pointerup` on `window`), position clamped to the viewport in `left`/`top` pixel state. **Gotcha for automated testing:** Chrome DevTools/`claude-in-chrome`'s synthetic `left_click_drag` only dispatches mouse events, not real `PointerEvent`s — a real user's mouse/trackpad fires both natively so dragging works fine for actual users, but automated drag-testing of this panel needs `dispatchEvent(new PointerEvent(...))` directly (via a `javascript_tool`-style page-context eval) to actually trigger the drag handlers.
- **Corner-clip gotcha (fixed, applied to both maps):** the map's rounded-corner clip must live on an INNER wrapper around just the `MapContainer` (`absolute inset-0 overflow-hidden rounded-box border ...`), not on the same outer `relative` container that also holds the floating panels. Putting `overflow-hidden rounded-box` on the shared outer container clips across the panel's own square top-left corner where the container's curve cuts into it — a visibly broken/jagged corner on the panel itself, easy to miss until someone screenshots it zoomed in.

## Image hosting (Cloudinary)
- All images are served from **Cloudinary** — cloud `dhvvz91bh`, asset folder `odyssey`. `public/art/` is **git-ignored** (kept locally only as the upload master / for re-sync); it is NOT deployed. `public/hero.jpg` stays in the repo (LCP preload).
- App delivers via `cldUrl(a, w)` in `App.tsx`: `…/image/upload/f_auto,q_auto,c_limit,w_${w}/${a.cld}` (AVIF/WebP, auto quality). Covers use `w_800`, lightbox `w_1600`.
- **Credentials:** `CLOUDINARY_URL` lives in `~/Documents/wallpapers-1850s/.env`; the `cld` CLI is in that project's `.venv` (`wallpapers-1850s/.venv/bin/cld`). This env uses **dynamic folders**, so uploads get *random* public_ids — the filename is stored as `display_name`.
- **Re-sync after adding/changing images:**
  1. `cd ~/Documents/wallpapers-1850s && export CLOUDINARY_URL=$(grep ^CLOUDINARY_URL= .env | cut -d= -f2-)`
  2. `.venv/bin/cld sync --push ~/Documents/odyssey/public/art odyssey -w 16`
  3. Map `display_name → public_id` via `cloudinary.search` (`expression("asset_folder=odyssey")`, paginate `next_cursor`) and write the `cld` field into `art.json` (see the one-off script used in git history).
- **Swap an image in place (no `art.json` change):** to replace an asset's bytes while keeping its `cld`/URL, overwrite by public_id: `cloudinary.uploader.upload(file, public_id=<cld>, overwrite=True, invalidate=True)`. The delivery URL stays valid and CDN is purged; pin a `/v<version>/` in the app's URL to force browsers past their cache (see `MAP_VER`). Used to swap the journey-map base. No orphan is created.
- **Orphan cleanup after removals:** removing an image from `art.json` leaves its Cloudinary asset unused. To delete orphans: list all `public_id`s in `asset_folder=odyssey` (paginated search), subtract the set of `cld` values still referenced in `art.json`, then `cloudinary.api.delete_resources(orphans)` (batches of 100). Verify with `resources_by_ids` that 0 *referenced* ids went missing. The search index is eventually-consistent, so the folder count lags deletes by a few minutes — confirm via a delivery-URL 404 instead. **Requires explicit user OK each time** (mass cloud-delete is gated by the safety classifier).
- Shell here is **zsh** (no `mapfile`/bash array slicing) — use Python for montages/scripts.
